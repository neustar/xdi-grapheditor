Overview

About XDI Graph Editor

The XDI Graph Editor is a visualization tool that facilitates exploring and editing XDI graphs. It can visualize the XDI graph with circles and lines to represent its nodes and links. In this documents, we refer to node and link as element in general.

Quick Start

Step 1 - Creating a minimal graph by importing an XDI statement
  1. To do so, click on File and then Import XDI button and enter the following line: [=]!:uuid:1111/$is$ref/=alice
  2. Click the Graph It button and you will see a graph with 4 nodes and 4 links.
  3. Some things worth noting on the generated graph:
    • The node with white fill color is the common root.
    • The dashed red line is a relational statement. The grey ones are contextual statements.
    • Edges are labeled in lightgray while nodes are labeled in black.
  4. Note that you may enter many XDI single display line statements at once.
Step 2 - Select and Moving nodes around
  1. Click on a node or drag from an empty space on screen to select nodes
  2. Drag nodes to move them around
Step 3 - Add a few contextual nodes
  1. Click Node and then New Node....
  2. A window appears to enter the node's label. Enter =bob (or another context node label).
  3. Create another node labeled =bob<#email>.
Step 4 - Link 2 nodes
  1. Click Link and then New Link... to start creating a new link.
  2. Drag from the =bob node to the =bob<#email> node.
  3. After releasing it, a new link appears between the 2 nodes. Notice that the label of the link is automatically inferred from the nodes labels.
  4. For consistency, link the common root node to the =bob node.
Step 5 - Add a literal statement
  1. Create a new node labeled "bob@email.biz".
  2. Link the =bob<#email> node to this node.
  3. You'll see an intermediary node is automatically created and both links with proper & labels added. Also note that the "bob@email.biz" node is now a literal node (green stroke).
Step 6 - Add a relational statement
  1. Link the =alice node to the =bob> node.
  2. Click on the link you just created to select it. Then click Link and then Set as Relation.
  3. You'll see the link turn to a red dashed line: this is a relational arc.

Graph Management

Import a graph from XDI statements

To create a graph from XDI statements, click File, and then click Import XDI....In the popup window, paste the XDI statements in the text input area.

There are three checkboxes:

  1. Delete Current Graph: if checked, the current graph will be cleared before create the new graph.
  2. Join the current graph: if checked, the statements being imported will be added to the current graph.
  3. Fold Root: if checked, the root node in the graph being imported will be folded when put on the screen.

Tip: This function will avoid overwhelming elements when importing large graphs. Unfold Level 1 function can then be used to unfold only the direct descendant of nodes.

Export a graph to XDI statements

To export the current graph into XDI statements, click File and then click Export to XDI.... A new window will pop up that contains the statements.

Note: The Popup window may be block by the browser.

Export a graph to image file

To export the current graph as a PNG file, click File and then click Export to PNG.... A new window will popup that contains the PNG file as a screenshot of the current graph. The graph will be exported as displayed on the screen. Therefore, elements that have been folded or hidden will not be included.

Note: This function is only allowed in Chrome browser.

Clear the current graph on screen

To clear the graph on screen, click File and then click Clear Graph. All the elements on screen will be completely deleted. This is an undoable operation.

Temporarily store a graph in browser storage

Click File and then click Store Graph. A window will popup asking for a graph name. Enter a graph name and click OK.

The graph will be stored in the local browser storage and will not be deleted when the page is refreshed or the browser is closed.

Load a graph from browser storage

Click File and then click Load Graph. A window will popup asking for the graph name to load. The names of currently stored graph will be listed in the window. Type the name to load and click OK.

Note: The new graph will override the current graph on screen.

Clear all graph stored in browser storage

Click File and then click Clear Storage. A window will popup. Type "YES" and click OK to confirm clearing the storage.

Graph Editing

Select an element

Left Click once on the element. For iPad, Tap once with one finger on the element.

Select multiple elements by clicking

Press and hold Shift and Left Click once on each element to select.

Note: This function is not available on iPad.

Select multiple elements by dragging

Press and hold the Mouse Left Button at any empty space on screen. Then move the cursor and a rectangle will show up. Release the Mouse Left Button and element within the rectangle area will be selected.

For iPad, Tap and hold with one finger at any empty space on screen. Then move finger and a rectangle will show up. Release touch and element within the rectangle area will be selected.

Move elements

First select the elements to move and them press and hold the Mouse Left Button and Drag.

For iPad, Tap and hold with one finger on one of the selected elements and move finger.

If multiple elements are selected, they will be moved together.

Cut/Copy/Paste elements

First select the elements to cut, copy or paste. Then click Edit and click the operation to perform.

For the Paste operation, the new elements will be placed near the original elements.

Rename elements / Change the label of elements

First select the elements to rename. Click Edit and then click Rename Objects.... A new window will popup with a text area to type the new names. The current label will be displayed in the text area.

If multiple objects are selected, then multiple windows will popup for each element one by one.

Delete elements

First select the elements to delete. Click Edit and then click Delete Object. This is an undoable operation.

Copy the labels of elements

First select the elements, click Edit and then click Copy Label.... A new window will popup with all the label listed in a text area. The text area is selectable as normal text editor.

To close the window, click X or Done.

Undo/Redo operations

Click Edit and then click Undo or Redo.

All the past operations are recorded. The user can undo all the way to the first operation or redo to the last operation.

Node Operations

Create a new node at the center of screen

Click Node and then New Node... or press N key on keyboard. A new window will popup with a text area asking for the label for the new node. Enter the label in the text area and click OK. The new node will appear at the center of the screen.

Create a new node at selected position on screen

Press and hold Shift key and click on the position to create node. A new window will popup with a text area asking for the label for the new node. Enter the label in the text area and click OK. The new node will appear at the clicked position.

Note: The mouse must click on an empty space to perform the operation, rather than elements or windows.

Fold the subtree of nodes

First select the nodes to fold, and click Node and then Fold/Unfold. Or Double Click on one of the selected nodes to fold the subtree of each selected node.

This operation will hide the subtree of the node.

Note:The stroke of a folded node will be red, indicating its subtree is hidden. If the nodes to fold donâ€™t have subtree, then nothing will be changed on the graph.

Unfold the entire subtree of folded nodes

First select the nodes to unfold, and click Node and then Fold/Unfold. Or Double Click on one of the selected nodes to unfold the subtree of each selected node.

This is the opposite operation of Fold the subtree of nodes. It will display the hidden subtree of the node.

Note: If the unfolded subtree contains other folded nodes, those nodes will remain folded.

Unfold only the direct descendant of a folded node

Click Node and then Unfold Level 1. Or press and hold Shift key and Double Click on the nodes to unfold.

This operation will only display the direct descendants of the nodes to unfold. Those descendants will become folded with their subtree hidden.

Tip: this operation can be used to explore large graph after Import a graph from XDI statements with Fold Root checked.

Unfold all the folded nodes in the graph

Click Node and then Unfold All.

This operation will unfold every nodes in the graph, including folded nodes in the subtree of other folded nodes.

Fix/Unfix the position of selected nodes under Force layout

First select the nodes to fix and then click Node and then Fix/Unfix or press F key.

The force directed layout constantly moving the nodes with force simulation. This operation will stop the layout from moving the fixed nodes as if they are pinned on the screen.

This is a toggle operation. If a node is fixed, it will be unfixed and vice versa.

Set a node as a root node

First select the nodes to set and the click Node and then Set as Root or press R key.

This operation will change the node type to Common Root. The syntax validator will check the syntax and report at the top left corner of screen.

The shape of nodes will change indicating the change of node type. This is a toggle operation. If a node is a root node, it will be change back to context node and its type will be determined based on its name.

Set a node as a literal node

First select the nodes to set and the click Node and then Set as Literal or press L key.

This operation will change the node type to Literal. The syntax validator will check the syntax and report at the top left corner of screen.

The shape of nodes will change indicating the change of node type. This is a toggle operation. If a node is a literal node, it will be change back to context node and its type will be determined based on its name.

Link Operations

Create a new link from one node to another

Click Link and then New Link... or press and hold Shift key. Then move cursor onto the source node, press and hold Mouse Left Button, move cursor to the target node and release the Mouse Left Button.

If the target node is a Literal Node and the source node is not a value node, a value node will be created between the source and target nodes.

The syntax validator will check the syntax and report at the top left corner of screen.

Invert the direction of a link

First select the link to invert and then click Link and then Invert Direction or press L key.

The syntax validator will check the syntax and report at the top left corner of screen.

Set link as a relation link

First select the link to set and then click Link and then Set as Relation or press R key.

The syntax validator will check the syntax and report at the top left corner of screen.

Graph Layouts

Switch graph layout

Click View and then select Force Layout or Tree Layout.

These are the two layouts to display the XDI Graph. The graph will transform to the new layout.

Reset the nodes to default positions under current layout

Click View and then Reset Layout

This operation will relayout the whole graph and move nodes back to the default position after layout.

Tip: this is useful especially in tree layout, after dragging nodes to different nodes.

Change the properties of current layout

The Layout Control panel is located as the top left corner of the screen. Drag the sliders to change the property values.

For each layout, there are some properties that control its details:

  • In Force Layout:
    • Link Distance: The length of links between nodes.
    • Link Strength: The hardness of links, which act like springs. The more rigid, the harder to stretch or squeeze and vice versa.
    • Node Repulsion:The force of nodes pushing against each other.
  • In Tree Layout:
    • Horizontal Margin: the horizontal margin between different depths in the tree.
    • Vertical Margin: the vertical margin between different branches in the tree.
  • In both layouts:
    • Node Size: The size of shapes that represent nodes in the graph.

Tip: Node Size is useful to prevent nodes from overlapping each other.

Reset the properties of current layout

Click View and then click Reset Layout. The parameters will be reset to default value.

Fix the positions of all nodes under Force layout

Click Fixed button on the Legend located at the left-bottom corner of screen.

The Force layout updates the nodes position with an global animation. This operation will stop all the nodes from moving

Unfix the positions of all nodes under Force layout

Click Moving button on the Legend located at the left-bottom corner of screen.

This operation will make the nodes start moving again after fix their position. Those nodes fixed with selection will remain fixed until they are selected and unfixed.

View Control

Zoom in and out

Press and hold option key on Mac and alt key on PC, and scroll mouse wheel to zoom in and out.

To zoom in/out by step, click View and click Zoom In or Zoom Out. Or press + or - key for zooming it or out.

The cursors will change to magnify glass icon to indicate the current graph. The boundaries of viewport and graph and the current scale are displayed in the Navigator at the bottom left corner of screen.

Zoom and fit the whole graph to window size

Click View and then Zoom to fit or press 9 key on keyboard.

The graph will be zoomed to fit into the screen with transition. The boundaries of viewport and graph and the current scale are displayed in the Navigator at the bottom left corner of screen.

Zoom and fit part of graph to window size

First select the elements, click View and then Zoom to fit or press 9 key.

The graph will be zoomed to fit only the selected part into the screen with transition.

Pan the viewport

Press and hold option key on Mac and alt key on PC, and Drag to pan the view.

The cursor will become hand icon to indicate the panning operation. The boundaries of viewport and graph and the current scale are displayed in the Navigator at the bottom left corner screen.

Reset zoom scale

Click View and then Actual Size or press 0.

The viewport will set back to zooming scale of 1 with transition. The boundaries of viewport and graph and the current scale are displayed in the Navigator at the bottom left corner screen.

Overview the graph with Navigator

In the navigator, the dark grey area indicates the boundary of graph, which is updated in real-time as the graph move. The grey rectangle is the boundary of viewport.

When the graph is smaller than the screen, the navigator will indicate the current position of graph on screen. When the graph is larger than the screen, the navigator will indicate the relative position of the current viewport to the whole graph.

Quick panning with Navigator

Move the cursor onto the navigator and drag to panning the graph.

Tip: this is a faster way of panning than panning in the graph.

Hide graph components

A legend is located at the bottom left of the screen. Click on an item to hide the corresponding graph components. The transparency of item indicates the visibility of graph component.

Search elements with keyword

Enter search queries

Click on the Search Bar on the top right corner and type the query. The Search Bar will expand into a panel and display the results.

The results will be updated as the query is entered. The current search is a string-match search with all the labels in the graph including the ones hidden with legend.

Note: The labels in folded subtrees are not included.

Browse search results

After The match nodes and links are displayed in two scrollable lists. The amount of matches is on the head of each list.

The color-coding of items are the same as elements in the graph. The literal nodes are green and relations are red.

Browse matched elements in the graph

The matched elements in the graph are highlighted with light blue. Their labels are displayed even if they are set to hidden in the Legend.

Navigate to match elements

Click on the item in the search lists and the graph will be zoomed to the corresponding element.

Close Search Panel

Click Done next to the Search Bar to close the Search Panel. The search text area and the lists will be cleared.

Customize windows visibility

Show/Hide a particular window

Click Window and then click the corresponding window to toggle visibility. The tick icon before each item indicates the visibility of the corresponding window.

Show/Hide all the window

Click Window and then click Show All or Hide All to show/hide all the windows together.

Advanced operations

Interaction Modes

This is a summarize of the modifier shortcuts described above. The XDI Graph Editor has three interaction modes and the user can press and hold modifier keys to switch modes.

  1. Browse Mode: drag, select and perform operations on elements in this mode. This is the default mode without any modifier keys held.
  2. Viewport Change Mode: Any viewport changes are performed in this mode. Current viewport operations are panning and zooming. Press and hold option key on Mac and alt key on PC to enter this mode.
  3. Edit mode: Add elements to the graph. Current adding operations includes creating new nodes and new lines. Press and hold Shift key to enter this mode.

Mouse Functions

With your mouse, you can do the following:
  1. Left-click: Select & Drag a new node.
  2. Shift + Click on Canvas: Create a new node.
  3. Shift + Drag from one node towards another: Add a link between the 2 nodes.
  4. Double-click a node: Fold the subtree of the node.

Node related keys

Once a node is selected (by clicking once on it), the following keys are enabled:
  1. R: Switch a node's type to Root. Note that common or inner roots are not distinguished. If the selected node was already of Root type, it will be changed back to Context.
  2. F: Fix/Unfix the movement of a node.
  3. Enter: Rename the node.
  4. Delete/Backspace: Delete the node along with all attached links.

Link related keys

Once a link is selected (by clicking once on it), the following keys are enabled:
  1. R: Switch a link's type to Relationship. If the selected link was already of Relational type, it will be changed back to Context.
  2. L: Inverse a link's direction.
  3. Enter: Rename the link.
  4. Delete/Backspace: Delete the link.

Viewport Control

Press and hold option/alt key, you can do the following:
  1. Drag: Pan the viewport.
  2. Scroll Mouse wheel: Scale the viewport.
A navigator at the bottom right indicates the size and position of viewport.

Search

There is a search bar at the top right. You can perform keyword search on nodes and links:
  1. Make a query: Click on the search bar and type your query. The results will be listed and updated as you are typing.
  2. Browse Results: The names of matches are listed. The lists are scrollable. All matches are highlighted in the graph.
  3. See it in graph: Click on an item and the view will pan and zoom to the node or link.
  4. Done: Click on done to hide the list.
