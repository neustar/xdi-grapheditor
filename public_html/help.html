Mouse Functions

With your mouse, you can do the following:
  1. Left-click: Add a new node.
  2. Shift + Click & Drag a node: Move a node to fixed position.
  3. Double-click a node: Free a fixed node from its position (i.e. allow it to float & position itself relatively to other nodes in the graph).
  4. Drag one node towards another: Add a link between the 2 nodes. It is recommended to fix the nodes' locations (see above) before attempting to link them (otherwise you'll have to chase the node...)

Node related keys

Once a node is selected (by clicking once on it), the following keys are enabled:
  1. R: Switch a node's type to Root. Note that common or inner roots are not distinguished. If the selected node was already of Root type, it will be changed back to Context.
  2. Enter: Rename the node.
  3. delete: Delete the node along with all attached links.

Link related keys

Once a link is selected (by clicking once on it), the following keys are enabled:
  1. R: Switch a link's type to Relationship. If the selected link was already of Relational type, it will be changed back to Context.
  2. L: Inverse a link's direction.
  3. Enter: Rename the link.
  4. delete: Delete the link.

Tutorial

Introduction

This page walks you through a few steps to create a simple graph and illustrate some of the major features of this tool.

Steps

Please execute the following steps:
  1. Step 1 - Creating a minimal graph by importing an XDI statement
    To do so, click on the "Import XDI" button and enter the following line: [=]!:uuid:1111/$is$ref/=alice
    Click the "Graph It" button and you will see a graph with 4 nodes and 4 links.
    Some things worth noting on the generated graph:
    1. The node with the orange stroke is the common root.
    2. The dotted red line is a relational statement. The grey ones are contextual statements.
    3. Edges are labeled in blue while nodes are labeled in black.
    Note that you may enter many XDI single display line statements at once

  2. Step 2 - Moving nodes around
    Press the Shift key, select a node (any node) and drag it around.
    You'll see it turn to red to indicate that its position is now fixed (or limited in movement). To release the node (i.e. allow it to move freely within the overall simulation) double click on the node. Doing so, will return the node's colour to grey (note: the colour change is buggy at the moment).

  3. Step 3 - Add a few contextual nodes
    Click in the window (away from any node or link).
    A window appears to enter the node's label. Enter =bob (or another context node label).
    Create another node labeled =bob<#email>.

  4. Step 4 - Link 2 nodes
    Fix the =bob node by clicking once on it. Do the same on the =bob<#email> node.
    Fixing those nodes will make the linking easier (this is not required though).
    Now that the nodes are fixed, you can place them anywhere you want by Shift-Click & Drag them.
    Click & drag the =bob node to the =bob<#email> node.
    After releasing it, a new link appears between the 2 nodes. Notice that the label of the link is automatically inferred from the nodes labels. To confirm the link, simply click on one of the nodes.
    For consistency, link the common root node to the =bob node.

  5. Step 5 - Add a literal statement
    Create a new node labeled "bob@email.biz".
    Link the =bob<#email> node to this node.
    You'll see an intermediary node is automatically created and both links with proper & labels added. Also note that the "bob@email.biz" node is now a literal node (green stroke).

  6. Step 6 - Add a relational statement
    Link the =alice node to the =bob> node but before clicking on a node (i.e. while the link is still a dashed line) press the "R" key.
    You'll see the link turn to a red dashed line: this is a relational arc.
